# シラバス作成ガイドライン

本ガイドラインは、大学数学カリキュラム・ロードマップに準拠したシラバスを一貫したスタイルで作成するための規則をまとめたものである。
---

## 1. 全体構成

### 1.1 ファイルヘッダー

各シラバスの冒頭は以下の形式とする。

```
# X-Y: コース名

**前提科目:** X-A（コース名）、X-B（コース名）

---
```

前提科目の ID とコース名は `roadmap_overview_detail.md` の記述に厳密に従う。

### 1.2 章・節・サブ項目の階層

```
## Chapter N: 章タイトル
**テーマ:** 章の主題（一文）

### §N.M セクションタイトル
**内容:** 扱うトピックの要約（一文）

- **1. サブ項目タイトル**
  - [アイテム群]

- **2. サブ項目タイトル**
  - [アイテム群]
```

- 章番号は `Chapter 1, 2, ...`
- 節番号は `§1.1, §1.2, ...`（章番号 + 節内連番）
- サブ項目は各節内で `**1.**, **2.**, ...` の太字番号つき見出し
- **各節は必ず2つ以上のサブ項目を持つ**こと

---

## 2. アイテムの種類と記法

### 2.1 使用できるアイテム一覧

| 種別 | 番号 | 用途 |
|---|---|---|
| Definition | あり | 概念・対象の厳密な定義 |
| Theorem | あり | 重要な結果（証明が非自明なもの） |
| Proposition | あり | 中程度の結果（定義から比較的直接導かれるもの） |
| Lemma | あり | 定理の証明に用いる補題 |
| Corollary | あり | 直前の定理・命題から直ちに導かれる系 |
| Axiom | あり | 公理（公理的集合論・公理的確率論など） |
| Example | あり | 具体例・計算例・反例 |
| Proof | **なし** | 直前の Theorem / Proposition / Lemma / Corollary の証明 |
| Remark | **なし** | 補足・注意・直感・応用の指摘 |

番号の形式は `種別 N.M-K`（例：`Definition 1.2-3` は §1.2 の3番目の Definition）。

### 2.2 記法の基本形

```
  - Definition N.M-K (内容の記述)
  - Theorem N.M-K (命題の主張)
  - Proof (証明のスケッチ)
  - Example N.M-K (具体例の内容)
  - Remark (補足・注意)
```

括弧内は日本語で記述する。数式は最小限にとどめ、散文を優先する（詳細は §4 参照）。

## 3. 各アイテムの配置ルール

### 3.1 Proof の配置

**Theorem・Proposition・Lemma・Corollary の直後に必ず Proof を置く。**

```
  - Theorem N.M-K (定理の主張)
  - Proof (証明のスケッチ：主要ステップを言葉で記述する)
```

- Proof には完全な証明でなくてよい。証明の「核心となるアイデア」「主要なステップの流れ」「使われる道具」を言葉で記述する。
- Proof の内容が Remark と混在しないよう注意する。Proof は「なぜ成立するか」、Remark は「何を意味するか・どう使うか」を担当する。

### 3.2 Example の配置

**Definition・Theorem・Proposition・Lemma の後には、なるべくほぼすべてに Example を置く。**

特に以下の場合は Example を省略しない。

- 定義：具体的なオブジェクト（関数・集合・演算子など）の例を1〜2個
- 定理：定理の仮定を満たす具体的な場合での計算・確認
- 定理の反例：仮定が外れたときに結論が成立しない例（「なぜ仮定が必要か」を示すもの）
- 条件の確認：定義の条件が満たされる・満たされないことを手で確かめる例

Example は**手で追える計算**を含むことを原則とする。「確認」「計算」「導出」を含む文言を積極的に使う。

### 3.3 Remark の配置

Remark は以下の内容を担当し、冗長にならないよう1〜3文程度にまとめる。

- 定義の直感的な意味、歴史的背景
- 定理の仮定の必要性（反例への示唆）
- 他の定理・概念との関係、後の章へのつながり
- よくある誤解・注意すべき点
- 物理・工学・他分野との接続

## 4. 数式の使用指針

### 4.1 基本方針：散文優先

**定義・定理・命題の本文は原則として日本語の散文で記述し、数式はその中に最小限に抑える。**

数式が不可欠な場合（等式・不等式・積分の式など）はインライン数式（`$...$`）を使用し、大型の display 数式（`$$...$$`）は本文では避ける。数式が多くなる場合は Example や Proof の中に移す。

良い例：
```
  - Definition 1.2-1 (収束横座標：ラプラス変換が収束する半平面の左端の実部 $\sigma_0$)
```

避けるべき例：
```
  - Definition 1.2-1 (収束横座標：$\sigma_0 = \inf\{s \in \mathbb{R} \mid \int_0^\infty |f(t)|e^{-st}dt < \infty\}$)
```

### 4.2 Example での数式使用

Example では計算過程を示すために数式を使用してよい。ただし複数行にわたる display 数式は避け、インライン数式にセミコロン `;` で区切って流れを示す。

```
  - Example 3.1-1 ($f(x) = e^{-|x|}$ のフーリエ変換：$x > 0$ と $x < 0$ に分けて積分すると
    $\hat{f}(\xi) = 2/(1+\xi^2)$；$\xi = 0$ での値は $\hat{f}(0) = 2$ で $f$ の総面積と一致)
```

### 4.3 Proof での数式使用

Proof では証明の核心ステップを数式で示してよい。ただし本格的な計算は省略し、「～を代入すると」「～と置換すると」などの言葉で流れを示す。

## 5. Example の質的要件

### 5.1 具体性の要件

Example は以下のいずれかを満たすこと。

1. **数値計算例**：具体的な数値・関数を代入し、答えを数値・式で示す
2. **検証例**：定義の条件や定理の結論を具体的なオブジェクトで確かめる
3. **反例**：仮定が外れたとき結論が崩れることを示す具体的なオブジェクト
4. **比較例**：2つ以上のオブジェクトを並べ、性質の違いを示す
5. **応用例**：定理・定義を物理・工学・他の数学分野に適用する

「～は重要な例である」「～などが挙げられる」のような抽象的な例の列挙は Example として不十分。必ず「確認」「計算」「検証」を伴う記述にする。

### 5.2 Example の粒度

- 1つの Example に盛り込む内容は「1つの計算の流れ」または「1つの比較」とする
- 複数の計算や比較を扱う場合は、Example の番号を分けて別アイテムにする
- 計算の結果を「～となる」「～が確認できる」で締める

## 6. サブ項目の分割ガイド

### 6.1 分割の原則

各節が単一のサブ項目にならないよう、以下の方針で分割する。

| 節の内容 | 分割の方法 |
|---|---|
| 定義が2つ以上あるとき | 「定義グループA」「定義グループB」に分割 |
| 定理が重要で証明が長いとき | 「定理の主張」「定理の応用・例」に分割 |
| 定義 → 定理 → 応用の流れがあるとき | 「定義と基本性質」「応用と発展」に分割 |
| 2つの独立した概念を扱うとき | 概念ごとにサブ項目を分ける |

### 6.2 サブ項目タイトルの書き方

サブ項目の太字タイトルは「何を扱うか」が一目でわかる名詞句とする。

良い例：
- `**1. ガウス関数（自己相似性）**`
- `**2. 矩形波と sinc 関数**`
- `**1. 測度の定義**`
- `**2. 基本的性質**`

避けるべき例：
- `**1. 定義と性質**`（何の定義か不明）
- `**1. その他**`（内容が不明）

## 7. 内容面のガイドライン

### 7.1 前提科目との整合性

- 前提科目で扱った定理・定義は「使う」ことを明記するが、再証明しない
- 参照の書き方：`（2-3 §X.Y を参照）` または `（実解析（2-3）の結果を利用）`
- 前提なしには理解できない議論を序盤に置かない

### 7.2 後続科目への伏線

- 後続科目へつながる定理・概念には Remark でその旨を示す
- 書き方：`（詳細は○○（X-Y）で扱う）` または `（○○（X-Y）への接続）`
- 過度に先取りして混乱を招かないよう、あくまで「予告」にとどめる

### 7.3 物理・応用との接続

- Stage 1〜3 の科目では、物理・工学的な動機や例を积極的に盛り込む
- 「なぜこの定義が自然か」「この定理はどの文脈で使われるか」を Remark で示す
- ただし厳密性を損なう形の「物理的議論」は Remark に限定し、本文に混入させない

### 7.4 反例・反証の配置

反例は以下の文脈で積極的に配置する。

- 仮定の必要性を示す（仮定を外したときに結論が崩れる例）
- 逆命題が成立しないことを示す
- 定義の境界事例（「ギリギリ条件を満たす」「ギリギリ満たさない」）
- 直感と異なる事実（反直感的な例）

### 7.5 証明に使う道具の明示

Proof では、使用する主要な道具・定理を明記する。

```
  - Proof (○○の証明：フビニの定理を使って積分の順序を交換する；以下 △△ の性質を適用する)
```

これにより学習者が「証明に何が必要か」を事前に把握できる。

## 8. 禁止事項・避けるべき記述

### 8.1 ラベルの禁止事項

- `Proof` に番号をつけない（番号なしが正しい）
- `Remark` に番号をつけない（番号なしが正しい）
- 番号付きアイテムに `Proof` と `Remark` を使わない

### 8.2 内容上の禁止事項

- 絵文字の使用禁止
- 非形式的な言い回し（「$n$ が大きくなると 0 に近づく」→「$n \to \infty$ のとき 0 に収束する」）
- 「手続き的注意」や「計算の注意事項」を Proposition や Theorem として扱う（Remark とする）
- 「自明」「明らか」「容易に確かめられる」などの記述（Proof に具体的な論拠を書く）

### 8.3 構造上の禁止事項

- 節内のサブ項目が1つだけになること（必ず2つ以上にする）
- Theorem / Proposition / Lemma の直後に Proof がないこと
- Definition の直後に Example が一つもないセクション

## 9. チェックリスト

シラバスの作成後、以下の項目を確認する。

### 構造チェック
- [ ] すべての節に2つ以上のサブ項目があるか
- [ ] 章・節の番号付け規則が一貫しているか（`§N.M`、`Definition N.M-K` など）
- [ ] `roadmap_overview_detail.md` の章構成と章タイトルが一致しているか

### Proof チェック
- [ ] すべての Theorem に Proof が続いているか
- [ ] すべての Proposition に Proof が続いているか
- [ ] すべての Lemma に Proof が続いているか
- [ ] Proof が番号なしになっているか

### Example チェック
- [ ] ほぼすべての Definition の後に Example があるか
- [ ] ほぼすべての Theorem / Proposition の後に Example があるか
- [ ] すべての Example に具体的な計算・確認が含まれているか
- [ ] 仮定の必要性を示す反例が適切に配置されているか

### 数式チェック
- [ ] 定義・定理の本文が散文中心になっているか（数式の過剰使用がないか）
- [ ] display 数式（`$$...$$`）が本文に不必要に使われていないか

### 内容チェック
- [ ] 前提科目の内容を再証明していないか
- [ ] 後続科目への接続が Remark で示されているか
- [ ] 物理・応用との接続が適切に示されているか（特に Stage 1〜3）
- [ ] コースのシラバスとして扱うべき主要概念が網羅されているか

## 10. サブ項目の典型的なパターン

以下に各種コンテンツの典型的な配置パターンを示す。

### パターン A：定義 → 例示型

新概念の導入に使う最も基本的なパターン。

```
- **1. ○○の定義**
  - Definition N.M-K (概念の定義)
  - Example N.M-K (定義を満たす典型的な例)
  - Example N.M-K (定義を満たさない例・反例)
  - Remark (定義の動機・直感的意味)
```

### パターン B：定義 → 定理 → 例示型

定義から帰結する性質を定理として述べるパターン。

```
- **1. ○○の定義と基本性質**
  - Definition N.M-K (概念の定義)
  - Example N.M-K (定義の確認例)
  - Proposition N.M-K (基本的な性質)
  - Proof (証明のスケッチ)
  - Example N.M-K (命題の具体的確認)
  - Remark (仮定の必要性・注意点)
```

### パターン C：複数定理の列挙型

1つのテーマに複数の定理が存在するパターン。

```
- **1. ○○定理**
  - Theorem N.M-K (定理の主張)
  - Proof (証明のスケッチ)
  - Example N.M-K (定理の適用例)
  - Remark (定理の意義・前提の意味)

- **2. △△定理**
  - Theorem N.M-K (定理の主張)
  - Proof (証明のスケッチ)
  - Example N.M-K (定理の適用例・比較)
```

### パターン D：定理 → 応用型

理論的な定理とその応用を分けるパターン。

```
- **1. ○○の定理（主張と証明）**
  - Theorem N.M-K (定理の主張)
  - Proof (証明のスケッチ)
  - Example N.M-K (定理の条件が満たされる典型例)

- **2. ○○定理の応用**
  - Example N.M-K (定理を使った具体的な計算)
  - Example N.M-K (別の応用・変形)
  - Remark (後続科目への接続・工学的意義)
```

## 付録：番号付け規則のまとめ

| アイテム | 形式 | 例 |
|---|---|---|
| Definition | `Definition N.M-K` | `Definition 3.2-1` |
| Theorem | `Theorem N.M-K` | `Theorem 1.3-2` |
| Proposition | `Proposition N.M-K` | `Proposition 2.4-1` |
| Lemma | `Lemma N.M-K` | `Lemma 4.1-1` |
| Corollary | `Corollary N.M-K` | `Corollary 2.3-1` |
| Axiom | `Axiom N.M-K` | `Axiom 1.2-1` |
| Example | `Example N.M-K` | `Example 5.2-3` |
| Proof | 番号なし | `Proof (...)` |
| Remark | 番号なし | `Remark (...)` |

ここで `N` は章番号、`M` は節番号、`K` はその節内での通し番号。同一種別ごとに独立にカウントする（Definition と Example の番号は別系列）。
