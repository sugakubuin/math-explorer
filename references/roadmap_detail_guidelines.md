# Roadmap Overview Detail 記述ガイドライン

`references/roadmap_overview_detail.md` を記述・加筆・修正する際のルールです。  
Stage に限らず、**全体で体裁と粒度が揃う**ようにしてください。

## 1. ファイル全体の構成（上から順）

次のブロックがこの順で並びます（自動生成時も同じ）。

1. **見出し1:** `# [詳細版] 大学数学 全分野学習ロードマップ`
2. **ステージごとの概要:** `## Stage-N：…` または応用分野のみ `## 応用数学系分野`
3. **各ステージのトピック一覧表**

   | ID | タイトル | 前提 |

   - 前提は複数トピック ID または `高校数学` などの文字列を **全角読点 `、`** で連結（例: `2-1、2-2`）。
4. 区切り線 `---`
5. **`## トピック数まとめ`** … ステージ別トピック数と **合計**（再生成で `roadmapData` と一致）
6. `---`
7. **`## 各トピックの章構成`** … 以降、トピックごとの詳細（次節のルール）

## 2. 「各トピックの章構成」ブロックの階層と字下げ

- **トピック:** `### [ID]: [分野名]`
  - 例: `### 4-1: 関数解析`、応用系は `### A-1: グラフ理論`
- **前提:** トピック見出しの直後に 1 行。`前提: …`（複数は `、` 区切り）
- **章:** `- **第[N]章: [章タイトル]**`（`[N]` はデータ上の章 ID に一致）
- **章の概要:** 章見出しの**すぐ次行**、字下げ **2 スペース**、ハイフン付き 1 行。章で扱うテーマを**簡潔に 1 行**（データの `description` に相当。省略する場合は生成スクリプトでは行自体が出ません）。
- **節:** 字下げ **2 スペース**、`- [章].[節]: [節タイトル] ([説明])`
  - 説明が空のときは括弧なしで `  - N.M: タイトル` だけになる場合があります。

## 3. 括弧（カッコ）の使い分け

実ファイルでは次のレイヤーを想定します。

- **節タイトル内の補足・別名（タイトルに含める部分）は「全角括弧（ ）」**
  - 例: `3.1: 外積（ウェッジ積）`、`4.2: 商群（因子群）`、`3.3: 次元公式（核-像定理）`
- **行末の説明文全体を囲むのは「半角括弧 ( )」**
  - 節タイトルの直後に**半角スペース 1 つ**空けて ` (説明文)` とします。
- **説明文の中に短い補足を入れる場合**は、読みやすさのため**内側に全角括弧**を使ってよいです（外側が半角のまま）。
  - 例: `(内積の定義、ノルム（大きさ）、なす角)`、`(定義と幾何学的意味（面積）)`

## 4. 説明文のスタイル（簡潔・明瞭）

- **過度な数式や冗長な文章を避ける**  
  厳密な定義式の羅列や長い叙述は避け、一覧性を優先します。
- **キーワード中心、または短いフレーズ**で「何を扱うか」が伝わるようにします。
  - *悪い例:* `(定数記号・関数記号・述語記号からなる形式言語 L の定義、項の帰納的定義)`
  - *良い例:* `(形式言語と項の帰納的定義)`
- **分野を象徴する記号・略号**は適度に残してよいです（例: `Ker`, `Im`, `L^p`, `ε-δ`, `grad` など）。冗長な式だけ削ります。

## 5. 表記のゆれに関する注意

- **見出しのコロン:** ステージタイトルはデータに合わせ、多くは **全角コロン `：`**（例: `Stage-1：教養としての数学`）。
- **トピック数まとめ:** ステージ列は `Stage-1` … `Stage-5` と **応用系**（自動生成時のラベル）。手修正する場合は本文中のトピック数と矛盾しないようにしてください。
